## 1. 产品背景与目标

### 1.1 背景
商家管理后台目前缺少统一的钱包管理能力。用户在平台内产生的充值、消费、退款、提现等资金行为缺乏可视化管理工具，运营人员无法实时追踪资金流向和处理异常。

### 1.2 目标
打造一套完整的钱包管理系统，覆盖：
- 钱包基础配置（开关、规则）
- 商户签约结算管理
- 充值方案配置与营销活动
- 用户钱包资金管理
- 资金流水追踪审计
- 充值/提现全流程管理

### 1.3 成功指标
- 运营人员可完成日常钱包配置操作，无需开发介入
- 一笔异常充值/提现可在 30 秒内定位到具体记录
- 提现流程全自动化率达到 80%（无需人工审核的占比）

## 4. 功能需求

### 4.1 钱包概览

**需求描述**：运营人员进入钱包模块后的首页，展示平台资金核心指标和近期趋势。

**页面内容**：

**4.1.1 统计卡片**
- 平台总余额、总充值金额、总消费金额、总退款金额
- 待审核提现笔数、钱包用户数、冻结账户数、今日充值金额
- 每张卡片显示环比变化百分比（绿色↑/红色↓）

**4.1.2 近 6 个月趋势图**
- X 轴：1月~6月
- Y 轴：金额
- 三条柱状系列：充值（蓝色）、消费（橙色）、退款（绿色）
- 柱状图自动适配最高值

**4.1.3 最近交易**
- 展示最近 8 条交易记录（流水号、用户、类型、金额、时间）
- 类型标签着色：充值蓝、消费橙、退款绿、提现红、冻结灰
- 金额正负颜色区分（收入红、支出绿）

**验收标准**：
- [ ] 柱状图随数据变化自适应高度
- [ ] 最近交易仅显示 8 条

### 4.2 钱包设置

**需求描述**：全局配置钱包功能开关及业务规则。

**4.2.1 编辑模式**
- **基础设置**：钱包总开关、钱包名称
- **充值设置**：最低充值金额、最高充值金额、收款商户（从商户列表选择）
- **提现设置**：提现功能总开关、是否需要人工审核（开关）
  - 手续费：固定金额/比例（%）
  - 提现方式：原充值订单退款（默认选中，不可切换）/ 主商户账户分账（置灰不可选）
  - 最低/最高提现金额、日提现限额、提现后余额下限

**4.2.2 查看模式**
- 只读展示当前配置
- 右下角「编辑」按钮 → 切换到编辑模式
- 编辑模式有「保存配置」「取消」按钮

**4.2.3 修改记录**
- 可折叠/展开的历史修改记录列表
- 每条记录可展开查看完整配置快照
- 记录字段：序号、修改人、修改时间、展开/收起操作

**验收标准**：
- [ ] 编辑/查看双模式切换正常
- [ ] 保存后数据持久化
- [ ] 修改记录正确展示历史快照

### 4.3 商户签约

**需求描述**：管理接入钱包的商户，支持签约/解约操作和收款流水查询。

**4.3.1 签约列表**
- 字段：商户名称、联系人、联系电话、签约状态、签约时间、解约时间
- 筛选：关键词搜索（商户名称/联系人/电话）、签约状态下拉

**4.3.2 新增/编辑签约**
- 商户选择：从商户列表下拉选择（编辑时不可更改商户）
- 联系人、联系电话、备注（全部选填）
- 签约协议：上传 PDF/JPG/PNG 文件（模拟，记录文件名）

**4.3.3 解约**
- 确认弹窗「确认与 xxx 解约？」
- 解约后状态变更为「已解约」，记录解约时间

**4.3.4 收款流水**
- 统计卡片：消费总额（蓝色）、退款总额（橙色）、净结算（绿色/红色）
- 分段控件：消费流水 / 退款流水
- 表格展示筛选后的交易记录

**验收标准**：
- [ ] 新增/编辑/解约功能完整
- [ ] 收款流水数据按商户过滤正确
- [ ] 解约后不可再操作

### 4.4 充值方案

**需求描述**：配置用户充值时可选的方案档次和营销活动规则。

**4.4.1 方案列表**
- 字段：方案名称、预选金额、活动数量、状态、生效时间
- 操作：编辑、详情、启用/禁用、删除

**4.4.2 方案编辑（弹窗）**
- 基础信息：方案名称
- 预选金额：可添加/删除/上移/下移金额选项，至少 1 个
- 自定义金额开关 + 最低/最高范围
- 充值活动列表（表格），可添加活动

**4.4.3 活动编辑（弹窗）**
- 活动名称
- 活动类型：赠送 / 折扣（分段控件）
  - 赠送：充值满 X 元，送 Y 元
  - 折扣：充值满 X 元，打 Y 折
- 活动时间范围（可选，不限表示永久有效）

**4.4.4 业务规则**
- 同一时间仅一个方案生效
- 启用新方案自动停用当前生效方案
- 生效中的方案不可删除
- 方案详情为只读弹窗

**验收标准**：
- [ ] 方案 CRUD 功能完整
- [ ] 启用/禁用/自动停用逻辑正确
- [ ] 活动编辑各个字段联动计算正确

### 4.5 用户钱包

**需求描述**：管理所有用户的钱包账户，查看资金明细，执行冻结/解冻操作。

**4.5.1 钱包列表**
- 字段：钱包ID、用户UID、手机号、余额、冻结金额、累计充值、状态、开通时间
- 筛选：关键词搜索（钱包ID/UID/手机号）、状态下拉（全部/正常/已冻结）

**4.5.2 资金流水弹窗**
- **顶部卡片**（两行布局）：
  - 行 1：用户信息卡片 + 状态卡片（含冻结/解冻开关）
  - 行 2：可用余额（本金蓝/赠送橙拆分）、消费金额（消费总额绿/已退款红）、冻结金额（系统冻结/提现中）
- **三段式切换**：充值流水 / 提现流水 / 交易流水
  - 充值流水：充值单号、支付单号、金额、状态、时间
  - 提现流水：提现单号、金额、关联单号、状态、时间
  - 交易流水：流水号、类型、金额、余额、时间（可展开子流水）

**4.5.3 冻结/解冻弹窗**
- 用户信息摘要（钱包ID、用户、当前余额、冻结金额、可操作金额）
- 冻结/解冻模式切换
- 金额输入 + 提示(可冻结/可解冻)
- 原因下拉选择 + 自定义补充
- 冻结流水历史列表

**验收标准**：
- [ ] 钱包列表按条件筛选正常
- [ ] 资金流水弹窗三选项卡数据正确
- [ ] 冻结/解冻操作更新余额和冻结流水

### 4.6 交易流水

**需求描述**：审计平台所有消费和退款交易记录。

**页面内容**：
- 字段：流水号、用户（UID+手机号）、商户、交易类型、金额、余额、关联单号、关联充值、时间
- 筛选：关键词搜索、交易类型（消费/退款）、商户名
- 操作：详情、导出

**详情弹窗**：
- 交易完整信息（10 个字段）
- 子流水：资金来源 FIFO 去向（消费时）/ 退款回充记录（退款时）

**数据范围**：仅消费和退款类型，充值/提现/冻结在其他模块

**验收标准**：
- [ ] 筛选器组合条件正确过滤
- [ ] 详情弹窗展示完整
- [ ] 子流水正确显示 FIFO 记录

### 4.7 充值管理

**需求描述**：管理充值订单，追踪支付状态和入账状态双维度。

**4.7.1 充值列表**
- 字段：充值单号、支付单号、用户、支付金额、到账金额、收款商户、支付状态、充值状态、申请时间
- 筛选：关键词搜索、充值状态下拉
- 操作：详情、查看流水、手动入账

**4.7.2 双状态管理**
| 支付状态 | 充值状态 | 说明 |
|---------|---------|------|
| 待支付 | 待入账 | 用户发起了充值但未支付 |
| 支付成功 | 待入账 | 已支付，入账处理中 |
| 支付成功 | 已入账 | 正常充值完成 |
| 支付失败 | 待入账 | 支付接口返回失败 |
| 已关闭 | 待入账 | 用户超时关闭 |
| 支付成功 | 入账失败 | 需人工处理 → 手动入账 |

**4.7.3 充值流水弹窗（资金桶视图）**
- 摘要卡片：用户、充值单号、支付单号、时间
- 本金桶 + 赠送桶（如有）
- 进度条：已用/剩余占比
- 消耗明细表：类型（消费/提现/退款）、关联单号、金额、时间

**验收标准**：
- [ ] 双状态追踪正确
- [ ] 手动入账功能生效
- [ ] 资金桶视图展示正确

### 4.8 提现管理

**需求描述**：提现审核与自动打款流程管理，支持人工审核和无需审核两种模式。

**4.8.1 提现列表**
- 字段：提现单号、用户、提现金额、审核方式、关联充值单号、关联支付单号、状态
- 筛选：关键词搜索、状态下拉
- 审核方式列：人工审核（黄色标签）/ 自动提现（蓝色标签）

**4.8.2 状态机定义**

`
状态流转：

                  ┌→ 通过 → 处理中 → 提现成功
待审核（需审核） ─┤                    └→ 提现失败 → 重新发起 → 处理中 → …
                  └→ 拒绝 → 已拒绝

待审核（无需审核）─→ 处理中 → 提现成功
                           └→ 提现失败 → 重新发起 → 处理中 → …
`

| 状态 | 含义 | 操作 |
|------|------|------|
| pending | 待审核 | 通过 / 拒绝（需审核）；自动执行（无需审核） |
| processing | 处理中 | 无操作（展示中） |
| success | 提现成功 | 无操作 |
| failed | 提现失败 | 重新发起 |
| rejected | 已拒绝 | 无操作 |

**4.8.3 审核操作**
- 点击「通过」→ 确认弹窗 → 状态变为 processing → 1.5 秒后模拟自动执行 → success 或 failed
- 点击「拒绝」→ 输入原因弹窗 → 状态变为 rejected
- 无需审核的记录：提交后自动执行（mock 直接显示结果）

**4.8.4 通知提示**
- 审核通过后提示「正在自动执行提现...」
- 自动执行成功后提示「提现单 xxx 自动提现成功」
- 自动执行失败后提示「提现单 xxx 提现失败，可重新发起」

**验收标准**：
- [ ] 需审核/无需审核两条路径都能走通
- [ ] 审核通过后自动执行且提示用户
- [ ] 失败后可重新发起
- [ ] 各状态标签颜色正确

## 6. 数据字典

### 6.1 实体关系

`
WalletConfig (1) ──── 全局配置

MerchantSign (N) ──── 商户签约信息

RechargePlan (N) ──── 充值方案
  └── RechargePreset (N)
  └── RechargeActivity (N)

UserWallet (N) ────── 用户钱包账户
  ├── RechargeRecord (N) ── 充值记录
  │     └── RechargeBucket (N) ── 资金桶
  ├── WithdrawRecord (N) ── 提现记录
  └── WalletTransaction (N) ── 交易流水
        └── BucketLog (N) ── 子流水
`

### 6.2 关键字段说明

#### WithdrawRecord 提现记录
| 字段 | 类型 | 说明 |
|------|------|------|
| status | enum | pending/processing/success/failed/rejected |
| needsReview | bool | true=需人工审核, false=自动提现 |
| refundType | enum | full/partial |
| originalPayMethod | enum | wechat/alipay/bank |
| payOrderNo | string | 关联支付单号 |

#### WalletTransaction 交易流水
| 字段 | 类型 | 说明 |
|------|------|------|
| type | enum | recharge/refund/consume/withdraw/freeze |
| bucketLogs | array | FIFO 核销明细，仅 consume/withdraw 有 |

## 7. 交互说明

### 7.1 通用交互规则
- 所有弹窗点击遮罩层可关闭
- 表格行 hover 有背景色变化
- 操作链接统一使用 .action-link 样式
- 状态标签统一使用 .status-tag 样式
- 金额列右对齐，使用等宽字体

### 7.2 反馈机制
| 操作类型 | 反馈方式 |
|---------|---------|
| 数据变更 | alert 提示 + 列表自动刷新 |
| 确认操作 | confirm 弹窗二次确认 |
| 拒绝提现 | prompt 弹窗输入原因 |
| 错误/失败 | alert 提示具体错误原因 |
| 文件上传 | 模拟选择后显示文件名 |
