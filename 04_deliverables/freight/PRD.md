## 1. 产品背景与目标

### 1.1 背景

商家在平台发布商品时，需要为商品配置运费规则。当前运费配置分散在商品发布页，缺乏统一管理能力，导致：

- 每个商品都要单独设置运费，重复工作量大
- 批量调整运费（如双11统一改包邮门槛）需要逐个修改商品
- 多种运费场景（如按件/按重量/金额阶梯/固定邮费）缺少统一模板支持
- 特殊地区加收规则无统一管理入口

### 1.2 目标

打造一套完整的运费模板管理系统，覆盖：

- 运费模板的增删改查
- 多种计费方式（固定邮费/按件计费/按重量计费/按金额阶梯）
- 默认运费规则 + 特殊地区规则
- 每条规则独立设置包邮条件
- 运费计算实时预览

### 1.3 成功指标

- 商家可在 2 分钟内完成一个运费模板的创建
- 一个运费模板可关联多个商品，批量生效
- 特殊地区规则地区冲突检测准确率 100%，不允许重复选中

### 4.1 运费模板列表

**需求描述**：展示所有运费模板，支持搜索和分页。

**页面内容**：

**4.1.1 搜索筛选**

**4.1.2 模板表格**

| 字段 | 说明 |
|------|------|
| 模板名称 | 模板的标识名称 |
| 计费方式 | 固定邮费/按件计费/按重量计费/按金额阶梯 |
| 默认规则 | 包邮状态 + 计费参数描述 |
| 特殊地区 | 特殊规则数量 |
| 操作 | 编辑 / 删除 |

**验收标准**：
- [ ] 搜索可按名称筛选
- [ ] 列表展示完整字段信息
- [ ] 删除有二次确认 + 商品关联保护

---

### 4.2 新建/编辑运费模板

**需求描述**：创建或编辑运费模板，配置所有规则。

#### 4.2.1 模板名称
- 文本输入框，必填
- 不超过 50 个字符

#### 4.2.2 计费方式（全局统一）
四种计费方式，切换时清空已有规则并弹窗确认：

| 计费方式 | 配置参数 | 说明 |
|---------|---------|------|
| 固定邮费 | 固定金额 | 每单统一收取固定邮费 |
| 按件计费 | 首件数 + 首费，续件数 + 续费 | 首件之后按续件单位递增 |
| 按重量计费 | 首重 + 首费，续重 + 续费（单位kg） | 首重之后按续重单位递增 |
| 按金额阶梯 | 多个金额区间（最低~最高 + 运费） | 按订单金额所在区间计费 |

#### 4.2.3 默认运费规则（全国兜底）
- 适用地区固定为「全国」
- 配置包邮设置 + 计费参数
- 不可删除

#### 4.2.4 特殊地区规则（覆盖默认）
- 可添加多条特殊地区规则
- 每条规则包含：适用地区、包邮设置、计费参数
- 地区选择以省份为单位，已占用的省份灰色禁选
- 多条规则配置后默认优先于默认规则

#### 4.2.5 包邮设置
每条规则独立配置：

| 状态 | 配置 |
|------|------|
| 不包邮 | 无额外参数 |
| 满额包邮 | 包邮门槛（金额/数量） |
| 无条件包邮 | 计费参数隐藏 |

包邮门槛与金额阶梯联动：有门槛时阶梯最后区间的 max 锁定为门槛值。

#### 4.2.6 金额阶梯（仅按金额阶梯时）
- 默认一个区间：0元~以上，运费 10 元
- 可添加多个区间，自动按中间值拆分
- 有包邮门槛时，最后区间 max = 门槛值且不可编辑
- 区间之间必须连续

#### 4.2.7 运费计算说明
按件/按重量计费时展示计算公式：

```
运费 = 首费 + ⌈(数量 − 首件) / 续件⌉ × 续费
当数量 ≤ 首件时，只收首费，不加续费
⌈⌉ 表示向上取整，除不尽时按一个续费单位计算
```

**验收标准**：
- [ ] 四种计费方式可正常切换，切换时弹窗确认
- [ ] 默认规则必须配置，不可删除
- [ ] 特殊规则可添加/编辑/删除
- [ ] 地区冲突检测正常工作
- [ ] 金额阶梯区间连续，联动包邮门槛
- [ ] 计算说明正确显示

---

### 4.3 删除模板

**需求描述**：只有无商品关联的模板可以删除。

**流程**：

```
点击删除
  ├── 有商品关联 → alert 提示「已被 N 个商品使用中，无法删除」
  └── 无商品关联 → confirm「未关联商品，确定删除？」
       └── 确定 → 删除成功
```

**验收标准**：
- [ ] 有商品使用的模板不可删除
- [ ] 无关联模板正常删除

## 6. 数据字典

### 6.1 实体关系

```
FreightTemplate (1) ──── 运费模板
  ├── defaultRule (1) ──── 默认规则（全国兜底）
  │     ├── regions: ['全国']
  │     ├── isFreeShipping: boolean
  │     ├── freeThreshold: number | null
  │     └── chargeConfig: 按计费方式不同
  │
  └── specialRules (N) ──── 特殊地区规则
        ├── regions: string[]
        ├── isFreeShipping: boolean
        ├── freeThreshold: number | null
        └── chargeConfig: 按计费方式不同
```

### 6.2 计费方式配置

#### 固定邮费 (FIXED)

| 字段 | 类型 | 说明 |
|------|------|------|
| fixedFee | number | 固定邮费金额 |

#### 按件计费 (BY_QUANTITY)

| 字段 | 类型 | 说明 |
|------|------|------|
| firstQty | number | 首件数 |
| firstFee | number | 首件运费 |
| additionalQty | number | 续件数 |
| additionalFee | number | 续件运费 |

#### 按重量计费 (BY_WEIGHT)

| 字段 | 类型 | 说明 |
|------|------|------|
| firstWeight | number | 首重 (kg) |
| firstFee | number | 首重运费 |
| additionalWeight | number | 续重 (kg) |
| additionalFee | number | 续重运费 |

#### 按金额阶梯 (BY_ORDER_AMOUNT)

| 字段 | 类型 | 说明 |
|------|------|------|
| amountRanges | AmountRange[] | 金额区间数组 |
| └ min | number | 区间最低金额 |
| └ max | number/null | 区间最高金额（null=以上） |
| └ fee | number | 该区间运费 |

### 6.3 运费计算规则

#### 按件/按重量
```
运费 = 首费 + ⌈(数量 − 首件(重)) / 续件(重)⌉ × 续费

数量 ≤ 首件(重)时，只收首费
⌈⌉ 向上取整，除不尽时按一个续费单位计算
```

#### 按金额阶梯
```
根据订单金额匹配所在区间，取对应运费
如有包邮门槛，订单金额 ≥ 门槛时免运费
```

#### 特殊地区优先级
```
买家下单 → 匹配特殊地区规则中的一个省份？
  ├── 是 → 用该特殊规则的运费 + 包邮条件
  └── 否 → 用默认规则的运费 + 包邮条件
```

## 7. 交互说明

### 7.1 通用交互规则
- 所有弹窗点击遮罩层可关闭
- 按 ESC 键关闭当前弹窗
- 弹窗内按 Enter 键确认
- 列表行 hover 有背景色变化

### 7.2 反馈机制

| 操作类型 | 反馈方式 |
|---------|---------|
| 切换计费方式 | confirm 弹窗确认 + 清空已配置数据 |
| 数据变更 | 弹窗内确定后更新展示 |
| 删除操作 | 无关联 → confirm 确认；有关联 → alert 提示 |
| 地区冲突 | alert 提示地区已被占用 |
| 保存模板 | alert 提示成功 + 跳转列表 |

### 7.3 模板编辑交互

```
新建/编辑页面
├── 模板名称输入
├── 计费方式选择（4 卡片）
├── 默认运费卡片 + 编辑按钮
│   └── 点击编辑 → 弹窗配置规则
│       ├── 适用地区（固定全国）
│       ├── 包邮设置（开关 + 门槛）
│       └── 计费参数（按类型动态展示）
│
└── 特殊地区规则区域
    ├── 规则列表（卡片展示）
    │   └── 每条含编辑/删除按钮
    └── 「+ 添加特殊地区」按钮
        └── 点击 → 弹窗配置规则
            ├── 地区选择（省份列表 + 冲突检测）
            ├── 包邮设置（开关 + 门槛）
            └── 计费参数（按类型动态展示）
```
